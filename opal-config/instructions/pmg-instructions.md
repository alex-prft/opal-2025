# PMG (Personalization Maturity Generator) Instructions

## Overview
The PMG system generates comprehensive personalization maturity assessments and implementation plans using a structured 4-phase framework: **Crawl → Walk → Run → Fly**.

## Core Principles

### 1. Scientific Assessment Methodology
- Use evidence-based scoring (1-5 scale) across 6 categories:
  - Data Foundation & Integration
  - Segmentation & Targeting Sophistication
  - Experience Personalization Breadth
  - Technology Infrastructure Maturity
  - Organizational Capabilities & Processes
  - Measurement & Optimization Practices

### 2. Phase-Based Progression Framework

#### CRAWL Phase (Foundation Building)
- **Duration**: 3-6 months
- **Focus**: Basic data collection, simple segmentation, email personalization
- **Success Criteria**: Consistent data flow, first personalized campaigns live
- **Key Technologies**: Email platform, basic CDP, simple analytics
- **Team Requirements**: 1-2 marketers, basic technical support

#### WALK Phase (Structured Growth)
- **Duration**: 6-9 months
- **Focus**: Web personalization, automated campaigns, journey optimization
- **Success Criteria**: Multi-channel personalization, measurable lift
- **Key Technologies**: Web personalization platform, marketing automation, advanced analytics
- **Team Requirements**: Dedicated personalization team, technical resources

#### RUN Phase (Advanced Execution)
- **Duration**: 9-18 months
- **Focus**: Real-time personalization, predictive analytics, omnichannel orchestration
- **Success Criteria**: AI-driven optimization, significant business impact
- **Key Technologies**: Real-time CDP, AI/ML platforms, comprehensive attribution
- **Team Requirements**: Cross-functional personalization center of excellence

#### FLY Phase (Mature Optimization)
- **Duration**: 18+ months
- **Focus**: Autonomous optimization, experience innovation, predictive experiences
- **Success Criteria**: Self-optimizing campaigns, industry-leading performance
- **Key Technologies**: Advanced AI/ML, IoT integration, emerging experience platforms
- **Team Requirements**: Innovation team, data scientists, experience designers

### 4. Data Integration Priority Hierarchy

1. **First-Party Data** (Highest Priority)
   - Website behavioral data
   - CRM/transaction data
   - Email engagement data
   - Mobile app interactions

2. **Zero-Party Data** (High Value)
   - Preference centers
   - Survey responses
   - Progressive profiling
   - Stated intentions

3. **Second-Party Data** (Strategic Partnerships)
   - Partner ecosystem data
   - Industry consortium data
   - Verified external sources

4. **Third-Party Data** (Supplemental Only)
   - Demographic enhancement
   - Interest/intent signals
   - Contextual data layers

### 5. Privacy-First Implementation

#### Consent Management
- Granular consent capture
- Preference persistence across channels
- Regular consent renewal processes
- Clear value exchange communication

#### Data Minimization
- Purpose limitation enforcement
- Retention period adherence
- Regular data purging processes
- Privacy-preserving analytics techniques

#### Transparency & Control
- Clear privacy notices
- Data usage explanations
- Individual access/deletion rights
- Opt-out mechanisms at any time

### 6. Success Measurement Framework

#### Phase-Specific KPIs

**Crawl Phase KPIs:**
- Data completeness rates (>60%)
- Email personalization lift (>5%)
- Segment coverage (>80%)
- Campaign deployment time (<1 week)

**Walk Phase KPIs:**
- Web personalization lift (>10%)
- Journey completion rate improvement (>15%)
- Cross-channel engagement lift (>20%)
- Time-to-insight (<24 hours)

**Run Phase KPIs:**
- Real-time personalization response (<100ms)
- Predictive model accuracy (>70%)
- Revenue attribution to personalization (>25%)
- Customer lifetime value improvement (>30%)

**Fly Phase KPIs:**
- Autonomous optimization coverage (>90%)
- Experience innovation pipeline (>5 experiments/month)
- Industry benchmark leadership (top 10%)
- Customer experience satisfaction (>90% NPS)

### 7. Technology Integration Guidelines

#### Optimizely Stack Integration
- **ODP**: Central customer data platform, ID resolution, audience management
- **Experimentation**: A/B testing framework, statistical significance validation
- **Content Recommendations**: AI-powered content optimization
- **CMP**: Campaign orchestration and performance tracking

#### External Integrations
- **CRM Systems**: Salesforce, HubSpot, Microsoft Dynamics
- **Email Platforms**: Marketo, Eloqua, Pardot integration via ODP
- **Analytics**: Google Analytics, Adobe Analytics, Mixpanel
- **E-commerce**: Shopify, Magento, BigCommerce personalization

### 8. Risk Mitigation Strategies

#### Technical Risks
- **Data Quality Issues**: Implement validation rules, regular audits
- **Integration Complexity**: Phased rollouts, fallback mechanisms
- **Performance Impact**: Load testing, caching strategies
- **Security Vulnerabilities**: Regular security assessments, encryption

#### Organizational Risks
- **Change Management**: Training programs, change champions
- **Resource Constraints**: Phased hiring, vendor partnerships
- **Stakeholder Alignment**: Regular communication, success showcases
- **Budget Overruns**: Milestone-based budgeting, ROI tracking

#### Compliance Risks
- **Privacy Violations**: Legal review, compliance monitoring
- **Regulatory Changes**: Adaptive consent framework
- **Data Breaches**: Incident response plans, insurance coverage
- **Audit Failures**: Documentation standards, regular reviews
